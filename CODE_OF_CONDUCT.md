# Code Of Conduct

This project is for developers, security teams, infrastructure engineers, and companies trying to adopt private AI safely. Collaboration should stay professional, specific, and useful.

## Expected Behavior

- Be respectful and direct.
- Assume contributors are trying to improve the project.
- Discuss technical tradeoffs with evidence.
- Keep security discussions responsible.
- Respect privacy and confidentiality.
- Avoid sharing real company data, secrets, or sensitive logs.

## Unacceptable Behavior

- Harassment or personal attacks
- Publishing private data or credentials
- Encouraging unsafe deployment practices
- Pressuring maintainers to bypass security review
- Spam or unrelated promotion
- Publicly posting exploit details before a responsible reporting path is available

## Enforcement

Maintainers may edit, hide, or remove issues, pull requests, comments, or contributors that violate this code of conduct. Serious security or privacy violations may result in immediate removal from project spaces.
